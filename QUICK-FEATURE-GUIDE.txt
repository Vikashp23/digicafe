✅ ADMIN FEATURES (admin.html)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. REQUEST MANAGEMENT TAB
   Location: Admin Dashboard → Sidebar → "Requests"
   
   View:
   ├── All client service requests in table format
   ├── Client name & email
   ├── Service type & amount
   ├── Request date
   └── Current status

2. CHANGE REQUEST STATUS (Dropdown)
   Status Options:
   ├── ⏳ Pending (Initial state)
   ├── ⚙️ In Progress (Work started)
   ├── ✅ Completed (Work finished)
   └── ⏸️ On Hold (Waiting for something)
   
   How to Use:
   └── Click dropdown in "Status" column → Select new status → Instant update

3. FILTER REQUESTS
   By Status:
   ├── All Status (view all)
   ├── Pending (only pending)
   ├── In Progress (only in progress)
   ├── Completed (only completed)
   └── On Hold (only on hold)
   
   By Service Type:
   ├── All Services (view all)
   ├── Income Tax
   ├── GST Registration
   ├── Company Reg
   └── TDS Compliance

4. UPLOAD DOCUMENTS
   Button: 📤 Upload
   How: Click button → Select files → Files attached to client's request
   Supports: PDF, DOC, DOCX, JPG, PNG, XLS, XLSX

5. DOWNLOAD DOCUMENTS
   Button: 📥 Download
   How: Click button → Client's files download to your computer
   Use: Get documents submitted by client for this request

6. REFRESH REQUESTS
   Button: 🔄 Refresh
   How: Click to reload latest requests from database
   When: Use after making changes or to see new submissions

✅ CLIENT FEATURES (index.html)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. UPLOAD DOCUMENTS SECTION
   Location: Client Dashboard → My Documents & Files
   
   Upload Card:
   ├── Click "Upload Documents" card
   ├── Select one or multiple files
   ├── Files upload instantly
   └── Appear in table below
   
   Supported Files: PDF, DOC, DOCX, JPG, PNG, XLS, XLSX

2. UPLOADED FILES TRACKER TABLE
   Shows:
   ├── File Name (with icon)
   ├── File Type (PDF, JPG, XLSX, etc.)
   ├── File Size (in KB)
   ├── Upload Date
   └── Delete Option (trash icon)
   
   Actions:
   ├── Delete: Click trash icon to remove file
   └── Keep: Files stay until you delete them

3. DOWNLOAD ALL FILES
   Button: 📥 Download
   How: Click to download all your uploaded files as bundle
   When: Get copies of documents you've submitted

4. MY APPLICATIONS TRACKER
   Location: Client Dashboard → My Applications
   
   Shows:
   ├── All services you've requested
   ├── Current status of each
   ├── Request date
   └── Amount paid
   
   Status Updates:
   └── Automatically updates when admin changes status

🎯 HOW TO USE — STEP BY STEP
───────────────────────────────────────────────────────────────────────────────

ADMIN WORKFLOW:
───────────────
1. Login to admin.html
2. Click "Requests" in sidebar
3. See table with all client requests
4. Change status using dropdown for each request
5. Upload documents for client using 📤 button
6. Download client documents using 📥 button
7. Use filters to find specific requests
8. Click Refresh to see latest updates

CLIENT WORKFLOW:
────────────────
1. Login to index.html
2. Go to Dashboard
3. Upload files using "Upload Documents" section
4. See uploaded files in table
5. Delete any files you don't need
6. Click "Download" to get all files
7. Check "My Applications" to see request status
8. Status updates automatically when admin changes it

❓ FREQUENTLY ASKED QUESTIONS
───────────────────────────────────────────────────────────────────────────────

Q: Can I change status back and forth?
A: Yes! Click dropdown anytime to change status

Q: Do files persist after refresh?
A: In demo: No (mock data). In real Firebase: Yes

Q: Can I upload same file twice?
A: Yes, system allows duplicates (you manage them)

Q: Will client see status changes immediately?
A: In real Firebase: Yes. In demo: After page refresh

Q: How many files can I upload?
A: Unlimited (subject to storage limits)

Q: Can I download individual files?
A: Demo: No (bundle only). Future: Yes

Q: Are files encrypted?
A: Demo: No. Real Firebase: Yes (automatic)

Q: Can clients see other clients' files?
A: No! Each client sees only their own files
